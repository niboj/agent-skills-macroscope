# Check-list de revue de skill

- [ ] `name` est valide et correspond au nom du dossier
- [ ] `description` explique quand utiliser la skill
- [ ] La skill couvre une seule unite de travail coherente
- [ ] Le chemin par defaut est clair
- [ ] Les chemins optionnels sont explicites, pas ouverts
- [ ] Les references ne sont utilisees que lorsqu elles apportent de la valeur
- [ ] Les scripts sont non interactifs et documentes
- [ ] Les etapes de validation sont concretes
- [ ] Les exemples sont realistes
- [ ] La skill est portable sauf si la dependance locale est intentionnelle
