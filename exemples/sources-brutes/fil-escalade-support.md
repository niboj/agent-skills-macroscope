# Exemple de fil source

Demande utilisateur :

"Nous demandons souvent a l agent de transformer des escalades brutes de support en courts resumes d incident avec impact, chronologie, cause racine probable et prochaine action. L agent ecrit souvent trop et melange les faits avec des suppositions. Je veux une skill reutilisable pour cela."

Corrections issues de la revue :

- Utiliser par defaut un resume en cinq sections.
- Separer les faits observes des hypotheses.
- Garder une chronologie sous forme de puces.
- Si aucun impact client n est connu, le dire explicitement.
- Ne pas inclure d etapes de remediation sauf si la demande le requiert.
