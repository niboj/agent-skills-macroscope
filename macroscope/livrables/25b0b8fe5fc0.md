####  Description
Le diagramme de cas d'utilisation permet de décrire comment le système sera utilisé. Il est composé d'un ou plusieurs cas d'utilisation qui spécifient les actions exécutées par le système et les résultats qui en découlent, et qui apportent une valeur ajoutée pour les parties prenantes du système. Il spécifie aussi les acteurs externes du système, ainsi que d'autres systèmes, qui interagissent avec le système modélisé. Les cas d'utilisation peuvent optionnellement être illustrés à l'intérieur d'un cadre délimitant les frontières du système.
####  Représentation graphique
Les descriptions d'éléments UML présentées ici sont basées sur le standard UML 1.5 [UML03] ou extraites de celui-ci. **Note** : Certaines des représentations graphiques ci-dessous contiennent des éléments grisés montrant le contexte, alors que l'élément UML faisant l'objet de la description est montré en noir. #### Lignes directrices orientées objet
Dans Solution, les diagrammes de cas d'utilisation peuvent être utilisés pour modéliser le contexte du système (P140S, P140O) ou pour fournir une représentation à haut niveau des processus du système (P201S, P201O). Voir la [table des stéréotypes UML de Solution](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_12701_99_60_NOT.html) pour la liste des stéréotypes pouvant être utilisés dans un diagramme de cas d'utilisation.
