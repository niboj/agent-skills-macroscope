####  Description
Un événement représente la conclusion d'une activité ou d'un processus (où les processus sont composés d'activités). Chaque activité entraîne le changement, pendant une période donnée, de ressources spécifiques en produits spécifiques par des agents spécifiques, selon les règles régissant l'activité. Par définition, l'événement représente un changement qui peut être observé, mesuré et dénombré. Une occurrence de l'événement se produit chaque fois que le résultat de l'activité correspondante est atteint.
####  Représentation graphique
#### Notation d'un événement (Modèle dynamique de processus OPAL)
Un événement est représenté par un cercle défini à l'aide d'une description et d'un identificateur (ID). La partie supérieure du cercle contient la description de l'événement, habituellement sous forme de nom suivi d'un participe passé indiquant le résultat de l'activité. La partie inférieure du cercle contient l'identification de l'activité ayant l'événement pour résultat. #### Lignes directrices générales
#### Description de l'événement
La description de l'événement indique le principal résultat atteint. La description de l'événement n'est pas celle de l'acte d'accomplir une chose. La description de l'événement doit être un substantif suivi d'un verbe au participe passé, par exemple : «Commande remplie», «Demande approuvée» ou «Client servi». Note : Un événement n'est pas seulement une fonction, tel que définie dans un diagramme de flux de données, dont on aurait changé la description pour un substantif suivi du participe passé d'un verbe. Par exemple, «Métiers gérés» décrit une fonction, mais non un événement. «Métiers gérés» ne pourra vraisemblablement pas être décrit comme un changement ni être observé, dénombré ou mesuré.
#### Identificateur de l'événement
L'identificateur de l'événement est un numéro unique qui sert à identifier l'événement. Aucune signification ne doit être rattachée à l'identificateur; on le déconseille parce qu'il est difficile de maintenir la cohérence d'un classement par hiérarchies ou par regroupements lorsque les modèles deviennent plus importants et reflètent mieux les processus d'entreprise représentés.  |
