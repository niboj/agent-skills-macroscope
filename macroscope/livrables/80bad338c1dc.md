####  Description
Conjointement avec les techniques de modélisation suivantes :
  * [P-OO-OC Classification et catégorisation des objets](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_173_60_39_TEC.html) ou [P-DP-EC Classification et catégorisation des entités](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_190_60_39_TEC.html);
  * [P-OO-UC Analyse détaillée de cas d'utilisation](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_181_60_39_TEC.html) ou [P-DP-UC Analyse détaillée d'une unité de tâche](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_191_60_39_TEC.html);
  * [P-OO-UO Analyse d'objet détaillée](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_179_60_39_TEC.html) ou [P-DP-UE Analyse d'entité détaillée](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_185_60_39_TEC.html),

cette activité décrit la structure d'ensemble de l'interface utilisateur, établit les aspects importants de la navigation, et détaille les spécifications des composantes clés et représentatives afin de permettre le prototypage des règles d'interface.
####  Notations
  * Les composantes d'interface peuvent être modélisées comme des classes orientées objet. Les diagrammes de classe résultants peuvent être organisés en regroupant les classes d'interface utilisateur en catégories. Typiquement, ces catégories correspondent aux couches de dialogue et de présentation et, le plus souvent, concordent avec les facettes du modèle d'information. Des catégories supplémentaires regroupent les composantes de base et les composantes partagées.
  * Pour les éléments complexes d'interface utilisateur, comme les documents avec une structure imbriquée de plusieurs niveaux, une certaine grammaire peut être utile. Les formes Backus-Naur (BNF) et les définitions de type de document SGML (DTD) sont des exemples de grammaires qui peuvent être utiles pour ce genre de spécifications.
