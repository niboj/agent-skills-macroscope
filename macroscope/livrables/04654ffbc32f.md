####  Raison d'être
  * Définir un composant logiciel en spécifiant soit les classes qui le constituent dans le cas de la modélisation orientée objet, soit les modules qui le composent dans le cas de la modélisation données-processus, ainsi que les interfaces applicables aux classes ou modules définis.
  * Les interfaces applicables à plusieurs composants logiciels peuvent être spécifiées séparément dans un P555S.


####  Contenu
**1. Description du composant logiciel** **2. Spécifications de composant logiciel O-O** 2.1 Classe, niveau réalisateur (une section par classe ou interface) **3. Spécifications de composant logiciel traditionnel (D-P)** 3.1 Module (une section par module et type de configuration d'infrastructure technologique) **4. Spécifications d'interface traditionnelle (D-P)** 4.1 Interface, niveau réalisateur (une section par interface, niveau réalisateur)
