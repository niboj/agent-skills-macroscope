####  Description
Le diagramme d'état montre des automates d'état incluant des états et des transitions. À l'intérieur de ces automates d'état, on peut définir le comportement d'un objet ou d'une interaction et décrire la séquence des états. "Typiquement, il est utilisé pour décrire le comportement de l'instance d'une classe, cependant, le diagramme d'état peut aussi décrire le comportement d'autres entités comme le cas d'utilisation, les acteurs, les sous-systèmes, les opérations ou les méthodes."[UML03]

Les descriptions d'éléments UML présentées ici sont basées sur le standard UML 1.5 [UML03] ou extraites de celui-ci. #### Lignes directrices orientées objet
Le diagramme d'état peut être est utilisé pour modéliser les états principaux des composantes du noyau (P170S) ou des composants de l'interface utilisateur (P490S, P487S, P186S), en conformité avec les modèles de processus. Il peut aussi être utilisé dans le [P555S Spécifications de composant logiciel](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_2670_99_34_DEL.html) (voir la section 2.1.4) pour illustrer les états possibles d'une classe.
