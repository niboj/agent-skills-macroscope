####  Description
Le diagramme d'objet montre une image du système à un moment précis. Il est surtout utilisé pour illustrer des cas spécifiques (exemples) et pour clarifier des aspects ne pouvant pas être représentés dans un diagramme de classe. Il représente des objets et les liens qu'ils ont entre eux. Un diagramme d'objet est une instance d'un diagramme de classe.
####  Représentation graphique
Les descriptions d'éléments UML présentées ici sont basées sur le standard UML 1.5 [UML03] ou extraites de celui-ci. **Note** : Certaines des représentations graphiques ci-dessous contiennent des éléments grisés montrant le contexte, alors que l'élément UML faisant l'objet de la description est montré en noir. La table suivante présente les éléments de notation pouvant être spécifiés dans un diagramme d'objet. ![](http://intradoc-cadrenormatif.itp.extra/DMRPFr/Images/FT_P_ObjectsDiagramNot1.gif) D'autres éléments de notation peuvent aussi être présentés dans un diagramme d'objet comme les attributs. Voir les [Éléments de notation généraux](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_12745_99_60_NOT.html) pour plus d'informations.
#### Lignes directrices générales
Dans Solution, le diagramme d'objet peut être utilisé comme complément au diagramme de classe. Voir le diagramme de classe pour plus d'information. Voir la [table des stéréotypes UML de Solution](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_12701_99_60_NOT.html) pour la liste des stéréotypes pouvant être utilisés dans un diagramme d'objet.
