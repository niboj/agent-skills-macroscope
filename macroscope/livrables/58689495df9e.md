####  Description
Un diagramme de classe représente la structure statique du système, indépendamment du facteur temps et du comportement à l'exécution. Il illustre les types d'entités (c.-à-d. les classes) qui font partie du système à modéliser, et les relations statiques qui existent entre elles.
####  Représentation graphique
Les descriptions d'éléments UML présentées ici sont basées sur le standard UML 1.5 [UML03] ou extraites de celui-ci. **Note** : Certaines des représentations graphiques ci-dessous contiennent des éléments grisés montrant le contexte, alors que l'élément UML faisant l'objet de la description est montré en noir. La classe est l'élément principal dans un diagramme de classe. Le tableau ci-dessous présente les éléments de notation qui lui sont associés. ![](http://intradoc-cadrenormatif.itp.extra/DMRPFr/Images/FT_P_DiagramNot1.gif) La table ci-dessous présente les types de relations pouvant être utilisés pour connecter les classes entre elles. ![](http://intradoc-cadrenormatif.itp.extra/DMRPFr/Images/FT_P_RelationshipsNot1.gif) ![](http://intradoc-cadrenormatif.itp.extra/DMRPFr/Images/FT_P_RelationshipsNot2.gif) Les interfaces qui sont offertes ou requises par une classe peuvent être illustrées dans un diagramme de classe. Les éléments de notation associés à la spécification d'interface sont présentés ci-dessous. ![](http://intradoc-cadrenormatif.itp.extra/DMRPFr/Images/FT_P_InterfacesNot1.gif) D'autres éléments de notation peuvent aussi être présentés dans un diagramme de classe :
  * des objets et leurs liens, qui sont le plus souvent présentés dans un diagramme d'objet. Voir le [Diagramme d'objet](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_12712_99_60_NOT.html) pour une description de ces éléments.
  * des paquetages et leurs relations, qui sont le plus souvent présentés dans un diagramme de paquetage. Voir le [Diagramme de paquetage](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_5160_99_33_NOT.html) pour une description de ces éléments.
  * des attributs, opérations, multiplicités, et d'autres éléments de notation généraux. Voir les [Éléments de notation généraux](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_12745_99_60_NOT.html) pour plus d'information.


#### Lignes directrices orientées objet
Dans Solution, des diagrammes de classe peuvent être utilisés pour modéliser le sujet (P200S, P150C) et ses facettes (P170S), la structure administrative des ressources (P201S, P239U), la structure de l'interface utilisateur (P250S, P176U), la structure d'un composant logiciel (P555S, P555C), et la structure des base de données (P510S, P460C, P510C). Voir la [table des stéréotypes UML de Solution](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_12701_99_60_NOT.html) pour la liste des stéréotypes pouvant être utilisés dans un diagramme de classe.
