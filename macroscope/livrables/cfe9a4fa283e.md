####  Description
Le prototypage des règles d'interface utilisateur permet principalement de vérifier leur pertinence et leur applicabilité. Il permet aussi de les compléter ou les raffiner, le cas échéant. On détermine d'abord la portée du prototype. Puis, basé sur les spécifications résultant de l'activité précédente, le prototype est ensuite réalisé, et comporte au besoin du code et des composantes non informatisées.
####  Extrants
Les livrables suivants peuvent être utilisés pour documenter les résultats de cette activité de technique. Les sections des livrables qui sont applicables sont montrées sous chaque livrable. Les livrables détaillés qui peuvent être utilisés au lieu des livrables succincts, ou en complément à ceux-ci, sont listés directement sous les livrables succincts concernés.
