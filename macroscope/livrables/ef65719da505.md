####  Description
Un diagramme de composant illustre la structure des composants (typiquement des composants logiciels) d'un système, et leurs relations avec les classes qu'ils réalisent et les artéfacts (par exemple fichiers de code, scripts) qui les implantent. Les classes expriment les exigences logiques des composants et sont définies dans un diagramme de classe. Similairement à un diagramme de classe, un diagramme de composant illustre les types de composant utilisés dans le système et non des instances de composants spécifiques. Le diagramme de composant est complémenté par le diagramme de déploiement qui montre comment les instances de composants logiciels seront déployées sur les nœuds de l'infrastructure technologique du système.

#### Lignes directrices orientées objet
Le diagramme de composant est utilisé dans le [P219S Architecture logicielle](http://intradoc-cadrenormatif.itp.extra/DMRPFr/html/Fr_P_1151_7_577_DEL.html) (section 1). Le diagramme de composant peut aussi être utilisé pour modéliser la décomposition du système en des sous-systèmes réalisateurs (P219S, P209C) et la structure du sous-système réalisateur (P219S, P219C).
