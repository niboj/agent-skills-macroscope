# Skill: creation de gabarits de skills

## But

Creer des gabarits reutilisables qui accelerent la creation de skills sans figer un domaine unique.

## Quand utiliser

- Quand un depot a besoin de gabarits standard pour de nouvelles skills.
- Quand des equipes veulent des fichiers de depart coherents pour rediger et revoir des skills.
- Quand des gabarits existants sont incoherents ou trop lies a un domaine.

## Ne pas utiliser

- Ne pas utiliser ce skill pour une seule skill finalisee, sauf si la sortie doit devenir un gabarit reutilisable.

## Entrees

- Les conventions du depot
- Le type de gabarit cible
- `references/catalogue-gabarits.md`

## Procedure

1. Lire `references/catalogue-gabarits.md`.
2. Choisir le plus petit gabarit qui repond au besoin de redaction vise.
3. Garder des placeholders explicites et faciles a remplacer.
4. Preferer une formulation portable a des exemples propres a un domaine, sauf si le gabarit est volontairement specialise.
5. Inclure uniquement les champs et sections utiles par defaut.
6. Ajouter une courte note d usage si le gabarit est susceptible d etre copie tel quel.

## Validation

- Le gabarit est reutilisable immediatement.
- Les placeholders sont evidents.
- Le gabarit n impose pas de fichiers inutiles.
- Le gabarit respecte les regles de nommage et de qualite du depot.

## Sortie

- Un ou plusieurs fichiers de gabarit reutilisables prets a etre ajoutes sous `gabarits/`.
