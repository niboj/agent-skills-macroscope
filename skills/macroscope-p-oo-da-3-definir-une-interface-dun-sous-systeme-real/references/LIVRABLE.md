## Synthèse du livrable

Cette activité de technique définit les interfaces de sous-système de niveau réalisateur. La définition des interfaces repose sur les dépendances décrites dans l'architecture logicielle. Une interface de sous-système définit les opérations publiques disponibles pour d'autres sous-systèmes. Il s'agit d'une façon d'exprimer le comportement d'un sous-système réalisateur sans exposer la structure interne (les classes réalisateur). Seules les opérations des classes en interaction avec les classes d'autres sous-systèmes sont exposées. Une interface de sous-système réalisateur peut être réalisée par plus d'un sous-système. Par ailleurs, un sous-système réalisateur peut être conforme à plus d'une interface de sous-système. Par conséquent, la relation entre un sous-système réalisateur et la définition de son interface n'est pas nécessairement biunivoque, mais chaque sous-système doit être identifiable à une définition d'interface de sous-système. Cette activité s'exécute en simultanéité avec l'activité P-OO-DA.2 Définir un sous-système réalisateur.
