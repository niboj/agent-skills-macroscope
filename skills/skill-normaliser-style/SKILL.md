# Skill: normalisation du style de skill

## But

Ameliorer la qualite d ecriture et la coherence entre skills sans elargir leur perimetre.

## Quand utiliser

- Quand plusieurs skills ont ete redigees par des personnes differentes et se lisent de maniere incoherente.
- Quand une skill est verbeuse, repetitive ou trop abstraite.
- Quand des conventions de depot doivent etre appliquees a des skills existantes.

## Ne pas utiliser

- Ne pas utiliser ce skill pour introduire un nouveau comportement.

## Entrees

- Un ou plusieurs fichiers `SKILL.md`
- Les conventions du depot
- `references/regles-style.md`

## Procedure

1. Lire `references/regles-style.md`.
2. Preserver la capacite et le perimetre de la skill.
3. Reserrer les sections vagues, repetitives ou generiques.
4. Remplacer les conseils abstraits par des instructions directes.
5. Aligner les titres, le style de check-list et le langage de validation sur le standard du depot.
6. Laisser le detail de reference propre au domaine hors du corps lorsque c est possible.

## Validation

- La skill reecrite est plus courte ou plus claire.
- Les etapes procedurales restent concretes.
- La description reste orientee declenchement.
- Aucun nouveau perimetre n est introduit.

## Sortie

- Un corps de skill normalise, accompagne d une courte note sur les principaux changements de style.
