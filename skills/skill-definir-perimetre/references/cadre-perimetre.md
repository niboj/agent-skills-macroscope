# Cadre de cadrage

Utiliser cette structure :

1. Probleme utilisateur repetitif
2. Artefact final souhaite
3. Workflow par defaut
4. Entrees requises
5. Modes d echec frequents
6. Non-objectifs
7. Skills adjacentes ou risque de chevauchement
8. Ressources de dossier recommandees

Regles de cadrage :

- Preferer un type de sortie stable par skill.
- Scinder la skill si le workflow bifurque tot vers des domaines differents.
- Garder les connaissances de domaine et les connaissances d evaluation dans des skills separees sauf si elles sont inseparables.
