# Motifs d instructions

Preferer ces motifs quand ils conviennent :

- Gabarit pour des formes de sortie fixes
- Check-list pour un travail multi-etapes dependant
- Boucle de validation pour les taches fragiles
- Planifier puis executer pour les operations destructrices ou en lot
- References conditionnelles pour le detail specifique a une variante

Supprimer ces anti-motifs :

- "Follow best practices" sans comportement concret
- Longues sections de contexte que le modele connait deja
- Grands menus d options sans chemin par defaut
- Exemples dupliques a la fois dans `SKILL.md` et `references/`
