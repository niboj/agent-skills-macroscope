## Synthèse du livrable

Les diagrammes comportementaux sont utilisés pour modéliser les différents aspects du comportement d'un système. Ils représentent le fonctionnement du système à travers un ensemble d'objets collaborant entre eux. La description du comportement peut prendre diverses formes : une interaction, un automate d'état (voir la définition anglaise à l'adresse suivante <http://www.nist.gov/dads/HTML/finiteStateMachine.html>), une activité, une opération. Le comportement du système est décrit comme une suite de changements apportés au système sur une période de temps. Les diagrammes comportementaux incluent le diagramme de cas d'utilisation, le diagramme d'état, le diagramme d'activité, le diagramme de séquence et le diagramme de collaboration. Un diagramme comportemental peut être attaché à un élément de modèle, par exemple à un cas d'utilisation, à une classe ou à une opération, afin de fournir une description de leur comportement.
