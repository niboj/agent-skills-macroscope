# Tableau de decision des ressources

- Le mettre dans `SKILL.md` si l agent en a besoin presque a chaque execution.
- Le mettre dans `references/` s il informe les decisions mais peut etre charge a la demande.
- Le mettre dans `scripts/` si la meme logique devrait sinon etre reecrite ou doit s executer de maniere deterministe.
- Le mettre dans `assets/` si le fichier est copie, transforme ou integre a des sorties.
- Le mettre dans `evals/` si vous voulez des cas repetables de type regression pour la skill.
