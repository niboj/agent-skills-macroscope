# Skill: Produire une vue d’architecture de service

## But

Assembler une vue cohérente à partir de plusieurs livrables Macroscope.

## Quand utiliser

- La demande dépasse un livrable unique.
- Il faut présenter une vue d’ensemble d’un service.
- Plusieurs domaines d’architecture sont concernés.
- Il faut produire une synthèse TOGAF exploitable.

## Entrées

- Demande utilisateur.
- Extraits, brouillons ou livrables existants, le cas échéant.
- `skills/INDEX.md` et `skills/variantes/INDEX.md` lorsque la sélection ou la comparaison de livrables est nécessaire.
- Références locales pertinentes dans les skills Macroscope.

## Procedure

1. Cadrer la vue : objectif, audience, décision à soutenir.
2. Sélectionner les livrables Macroscope contributeurs.
3. Identifier les éléments métier, données, applicatifs, technologiques, sécurité et migration.
4. Structurer la vue selon TOGAF.
5. Relier exigences, décisions, risques et composants.
6. Produire les diagrammes ou matrices nécessaires.

## Validation

- La sortie répond au besoin utilisateur sans élargir inutilement le périmètre.
- Les liens avec TOGAF sont explicites lorsque l’architecture est concernée.
- Les hypothèses, limites et points à confirmer sont visibles.
- Les chemins cités existent.
- Les livrables ou variantes recommandés sont justifiés.
